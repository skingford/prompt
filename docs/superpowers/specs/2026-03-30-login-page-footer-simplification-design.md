# 登录页底部轻量化设计

## 概述

本次工作目标是优化登录页视觉重心，移除底部的中英文切换控件，让页面更聚焦于品牌信息和登录表单本身。

这次改动只涉及登录页底部展示层，不调整认证流程，也不移除项目现有的国际化能力。

## 背景

当前登录页底部同时承载了：

- 中英文切换控件
- 隐私政策、服务条款、支持链接
- 版权文案

其中语言切换胶囊的视觉权重较高，会把注意力从表单区域拉向页面底部。对于以“快速登录”为唯一主任务的页面，这一层级分配不够克制。

## 范围

### 包含内容

- 移除登录页底部的语言切换入口
- 将登录页底部收敛为轻量页脚信息区
- 微调页脚与表单之间的间距、字重和视觉层级
- 保证桌面端和移动端宽度下布局稳定

### 不包含内容

- 移除项目中的国际化能力
- 调整工作台或其他页面中的语言切换入口
- 修改登录逻辑、表单字段或跳转流程
- 新增真实的隐私政策、服务条款、支持页面

## 方案对比

### 方案 A：移除语言切换，保留单层轻页脚（推荐）

做法：

- 去掉登录页底部 `LocaleSwitcher`
- 页脚仅保留链接行和版权文案
- 压缩底部占用高度，降低文字与边框的存在感

优点：

- 最符合登录页单一任务场景
- 视觉最干净，表单更突出
- 改动范围小，风险最低

缺点：

- 页面底部的装饰性会比现在更弱

### 方案 B：移除语言切换，保留胶囊式底部链接组

做法：

- 去掉语言切换控件
- 将底部链接做成一个较轻的胶囊容器

优点：

- 仍保留一定精致感

缺点：

- 底部仍有较明显的视觉块状结构
- 对“突出登录表单”这件事帮助不如方案 A 直接

### 最终选择

采用 **方案 A**。

原因：

- 用户已明确希望去掉登录页底部中英文切换
- 当前页面最重要的目标是快速完成登录，而不是提供额外偏好设置入口
- A 方案在不改变页面结构的前提下，能最直接地减少视觉噪音

## 设计说明

### 页面结构

登录页整体结构保持不变：

- 顶部品牌区
- 中部登录表单卡片
- 底部页脚信息区

变化仅发生在页脚信息区：

- 删除语言切换控件
- 底部链接独立成一行
- 版权文案独立成一行

### 视觉调整

页脚区应比当前更轻：

- 缩小与表单卡片之间的垂直间距
- 降低文字字号或强调度，避免与登录按钮竞争注意力
- 保留足够留白，避免底部显得拥挤

目标效果：

- 用户第一视觉焦点停留在标题、输入框和登录按钮
- 页面底部变成辅助信息，而不是操作控件区

## 实现边界

预计主要修改：

- `src/pages/LoginPage.tsx`
- `src/styles.css`

具体实现方式：

- 从登录页中移除 `LocaleSwitcher` 的引用和渲染
- 删除登录页专用的 `login-footer__locale` 布局依赖
- 调整 `login-footer`、`login-footer__links`、`login-footer p` 等样式，使其符合轻页脚节奏

说明：

- `LocaleSwitcher` 组件本身不删除，避免影响其他页面
- `i18n` 初始化逻辑和语言资源保持原样

## 风险与兼容性

风险较低，主要关注：

- 移除切换控件后，底部间距是否出现空洞
- 小屏宽度下页脚链接是否换行异常
- 登录页若依赖当前语言显示中文文案，需要确认默认语言仍按现有机制工作

## 验证方案

### 功能验证

- 登录页底部不再显示中英文切换
- 用户名、密码、忘记密码、登录、注册链接行为不受影响
- 登录成功后的跳转行为不变

### 视觉验证

- 桌面宽度下，页脚视觉重量明显低于表单区
- 移动宽度下，底部链接与版权文案对齐自然，不遮挡或挤压内容
- 页面整体重心较改动前更集中于表单卡片

## 验收标准

- 登录页中不再出现“中文 / English”切换控件
- 页脚仅保留辅助链接和版权文案
- 页面视觉层级更清晰，底部不再喧宾夺主
- 不影响项目现有国际化能力及其他页面行为
