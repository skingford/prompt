# Claude Code 风格拆解与参考文案（基于 Context7 + 站点抓取）

更新时间：2026-03-05

## 1) 设计风格总结

### 整体气质
- 工程工具感 + 品牌编辑感：不是纯技术文档风，也不是重营销风，而是“可直接开工”的产品展示页。
- 语气克制直接：强调“你描述目标，系统执行任务”，少空话，多动作动词（构建、调试、交付）。

### 视觉语言
- 主体为浅暖中性色背景，配深灰文字与 clay 橙色强调（示例变量：`#faf9f5`、`#141413`、`#d97757` / `#c96442`）。
- 大面积留白 + 高层级标题，阅读节奏偏“编辑排版”。
- 字体采用品牌化三件套：Sans（正文/界面）、Serif（强调/叙事）、Mono（代码/终端）。

### 布局结构
- Hero 直接给价值主张 + 终端演示。
- “工作场景”分块（终端 / IDE / 桌面 / Web & iOS / Slack）强化“在哪都能用”。
- 中后段用 pricing + FAQ 收口，降低决策摩擦。

### 动效策略
- 以过渡、淡入、分段 reveal 为主（GSAP/clip-path/stagger 类），动效服务信息层级，不喧宾夺主。

## 2) 主题（Theme）提炼

可概括为一句：

**“让 AI 在你现有工作流里执行复杂开发任务，而不是让你迁移到新工作流。”**

支撑主题的 4 个叙事支柱：
- 工作流原位集成：terminal/IDE/desktop/web/slack。
- 深度代码理解：跨文件、跨依赖、跨上下文。
- 从想法到提交：任务闭环（分析、修改、验证、交付）。
- 可控与可信：用户可审查、可中断、可管理权限。

## 场景区
- 小标题：在你工作的地方使用你的产品名
- 卡片1（终端）：在命令行中完成代码探索、批量修改与任务自动化。
- 卡片2（IDE）：以可视化 diff 审阅改动，边看边改，减少上下文切换。
- 卡片3（桌面）：并行处理多个任务，统一查看进度与结果。
- 卡片4（Web / Mobile）：离开工位也能发起任务、跟进执行状态。
- 卡片5（协作工具）：在团队协作渠道中触发自动化开发流程。

## FAQ
- 它能处理哪些任务？
  - 回答：从日常修复、重构到功能实现，均可在你的约束下执行并返回可审查结果。
- 是否兼容现有开发工具？
  - 回答：支持终端、主流 IDE 与协作工具，尽量不改变团队已有流程。
- 如何保证安全与可控？
  - 回答：所有关键操作可审查、可授权、可回滚，支持团队级治理策略。
